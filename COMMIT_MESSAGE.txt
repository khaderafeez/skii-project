feat: Integrate Polar H10 Bluetooth with real-time HR/HRV streaming

## Summary
Complete integration of Polar H10 heart rate sensor with React UI via Bluetooth and WebSocket.
System runs entirely on Raspberry Pi 3B+, streaming real-time HR and HRV (RMSSD) metrics with
automatic reconnection and connection status display.

## Changes

### Backend (New)
- backend/polar_server.py: Python WebSocket server (262 lines)
  * Connects to Polar H10 via Bleak
  * Streams HR + RR intervals every 2 seconds
  * Calculates RMSSD (HRV metric)
  * Auto-reconnects on Bluetooth disconnection
  * Broadcasts to all connected WebSocket clients

- backend/requirements.txt: Python dependencies
  * bleak==0.21.1 (Bluetooth)
  * polar_python==0.2.0 (Polar device)
  * websockets==12.0 (WebSocket server)
  * numpy==1.24.3 (Math)

- backend/setup.sh: Automated Pi setup script
  * Installs Python, Bluetooth libraries
  * Creates virtual environment
  * Provides Bluetooth pairing instructions

- backend/setup_systemd.sh: Systemd service setup
  * Registers backend for auto-start on boot
  * Enables service management via systemctl

- backend/polar-hrv-monitor.service: Systemd unit file
  * Configures Python backend as service

- backend/test_websocket.py: WebSocket test utility
  * Debug tool to verify backend connectivity

### Frontend (New)
- src/hooks/usePolarData.ts: React WebSocket hook (173 lines)
  * Connects to ws://localhost:8765
  * Auto-reconnects with exponential backoff
  * Handles connection timeouts
  * Returns: { heartRate, hrv, rr_intervals, isConnected, error }
  * Includes debug logging

### UI Updates
- src/pages/LiveMonitoringScreen.tsx (Modified)
  * Replaced mock data generation with usePolarData hook
  * Displays real HR/HRV from WebSocket
  * Shows real Bluetooth connection status
  * Removed hardcoded random value loops

- src/pages/SessionStartScreen.tsx (Modified)
  * Replaced mock HR simulation with real data
  * Shows "WAITING FOR DEVICE" when not connected
  * Displays actual error messages
  * Uses real HR for baseline measurement

## Key Features

✓ Real-time streaming: HR and HRV updates every 2 seconds
✓ Auto-reconnect: Automatic recovery from Bluetooth disconnections
✓ Connection status: Clear UI indication of device connection state
✓ Error handling: Graceful handling of missing device or connection failures
✓ Auto-start: Systemd service enables Pi boot startup
✓ Zero friction: Frontend automatically connects to backend
✓ No data persistence: Raw readings only (phase 2 for storage)
✓ Comprehensive docs: Complete setup and troubleshooting guide

## Data Flow

Polar H10 (Bluetooth)
    → Python WebSocket Server (ws://localhost:8765)
        → React Frontend (http://localhost:5173)
            → UI: Real HR (bpm), HRV/RMSSD (ms), Connection Status

## Testing

- Backend: `cd backend && python3 polar_server.py`
- WebSocket: `cd backend && python3 test_websocket.py`
- Frontend: `npm run dev`
- Expected: Real HR/HRV displayed in LiveMonitoringScreen

## Deployment

1. Setup: `cd backend && ./setup.sh`
2. Pair device: `sudo bluetoothctl`
3. Auto-start: `cd backend && ./setup_systemd.sh`

See QUICK_START_PI.md for detailed instructions.
